MTA_Cross_Schema

SAP HANA XS Advanced, create a user-provided anonymous service for SAP HANA database Schemas

This example app requires a uaa service instance.

Use the XS-A CLI to create one with the following command.

xs create-service xsuaa default cross-uaa
xs cups CROSS_SCHEMA_LOCATIONS -p '{"host":"hxe2sp02.sap-a-team.com","port":"30013","user":"SYSTEM","password":"Plak848!","driver":"com.sap.db.jdbc.Driver","tags":["hana"] , "schema" : "LOCATION_TEST" }'

CALL "CROSS_1"."grant_role"(
	IN_USER => 'INA_USER'/**/,

	EX_MESSAGE => ?
);

Another way is to create a new empty role (HDIACCESS) with the SYSTEM user and then to grant it to a user.
Then grant the cross_admin role to the HDIACCESS role.  This 2-level works where granting cross_admin directly to a user doesn't.
